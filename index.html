Call to Action is an open-source web platform for creating telephone-based services such as hotlines, voice petitions and phone blogging that grew out of a project at MIT's Center for Civic Media. The platform provides an easy-to-use graphical interface that enables the user to plan the flow of calls, record custom audio and make use of all the input and output features offered by a regular telephone. The service requires no software programming experience, and users can build a service in under half an hour. Our design process so far has been built on user-centered and co-design methodologies, working in partnership with the communities who've used similar services in the past.

We have released an alpha version of the platform's design interface, but now we need to develop the scripting and hosting functions the platform will use. That's why we're launching this bitstarter campaign. We know there are lots of use cases out there, and lots of talented engineers working on VoIP. So come and join us, and help make open source community VoIP projects an easy, lightweight reality for groups everywhere. Folks across the world have been implementing innovative voice solutions to social problems - in the future, Call to Action can help make some of those projects, faster, stronger and easier to maintain.

FAQ

Why are we developing this platform?
Following the launch of New Day New Standard, an innovative phone service the team developed with REV- and Domestic Workers United to inform domestic workers and their employers in New York of changes in legislation affecting them, it was clear that a wide range of community groups could benefit from developing phone-based information services of their own. Since NDNS was created in VoIP Drupal scripting language, though, users without that specific programming experience would need to invest a significant amount resources to create a similar service. Secondly, our research has found that most user communities find creating a voice tree to be a conceptually challenging exercise. We hope to simplify the process with a visually oriented system that mirrors a paper-and-pen workshop method the team uses with community partners.
How does the platform work?
Users build their services by dragging and dropping pre-defined elements onto a canvas and linking these elements with ties. Elements are distinguished by colour and shapes to help users to quickly identify the elements they need. Commonly used sequences of elements (such as a caller recording a message to be submitted to the service) are given as grouped templates or 'blocks' that can be imported onto the canvas. After completing a design, the user can save their work for publishing, and retain the files for future editing. As well as facilitating the easy creation of telephone services, the platform gives users advice on how to record effective voice prompts and estimates the call length at each stage of the call flow. This allows creators to optimize their services to suit caller needs.
How are you building Call to Action?
Call to Action uses Javascript (backbone) and CSS to handle the manipulation of elements on the canvas. Our goal is that the platform will output to a script that can be imported into several VoIP frameworks, including VoIP Drupal, Twilio and Tropo. The platform's working code is maintained on github, and you can read more about the background and the team at calltoaction.mit.edu.
